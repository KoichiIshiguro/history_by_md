# プロジェクト管理システム AI機能 — コンセプト提案

## 今困っていること

プロジェクト管理システムにMarkdownで書き溜めたノートが大量にある。ノートが増えるにつれて、「どこに書いたか」「いつ・なぜそう決めたか」を探し出すのが難しくなっている。

キーワード検索では、言い回しや表現が違うだけでヒットしない。自分が書いたことなのに、自分で見つけられない状態になりつつある。

---

## 実現したいこと

### 1. ノート全体を把握したAIへの質問

ノートの内容をAIが把握していて、自然言語で質問したら、過去の記録から該当する記述を見つけて正確に答えてくれる。キーワードの一致ではなく、意味として近いものを探せること。

**例**:

- 「○○プロジェクトの認証方式、なんでOAuthにしたんだっけ？」→ 当時の検討メモをもとに回答
- 「△△クライアントとの最後の打ち合わせで何を決めた？」→ 該当する議事メモから回答
- 「過去にXXと似た問題が起きたことある？」→ 関連する記録があれば提示

ノートが編集されたら、なるべく早く検索結果に反映されてほしい。

### 2. システム上でのAIチャット機能

既存のプロジェクト管理システム内にチャットUIを追加し、そこからAIに対して質問・やり取りができるようにしたい。ターミナル（Claude Code）からだけでなく、ブラウザ上から直接使えるようにする。

### 3. ブロック編集中のAI文章生成

ノートの編集中に、AIの力を借りて文章を生成・挿入できる機能がほしい。

**操作フロー**:

1. 未入力状態のブロックで先頭に半角スペースを入力すると、AIプロンプトモードが発動する
2. プロンプトを書いてEnterを押すと、AIが文章を生成する
3. 生成中はローディングエフェクトを表示し、キャンセルボタンも用意する
4. 生成が完了したら、モーダルで内容を確認させる
5. 確認OKで初めてブロックに挿入する

**キャンセルについて**:

- 明示的なキャンセルボタンのクリック、またはブラウザを閉じるなどの操作でのみ生成を停止する
- それ以外の操作では生成を中断しない

---

## 前提・制約

- AI処理はサーバー上にインストール済みのClaude Codeを使い、`claude -p`（非インタラクティブモード）で実行する
- ノート検索にMCPサーバーを活用したいが、これも`claude -p`経由で利用できるか確認したい。stdio方式のMCPサーバーは`-p`モードで動作することを確認済み。1リクエスト=1プロセスで完結するため、タスク待ちは発生しない想定
- 自社サーバーのスペックは高くないので、重い処理は外部サービスに頼りたい
- コストはできるだけ抑えたい。無料枠で収まるならそれが理想
- `claude -p`はプロセス起動に数秒かかるため、レスポンスは即座ではなく数秒の待ちが発生する

## 検討中の外部サービス

以下を候補として考えているが、より良い選択肢があればエンジニア側から提案してほしい。

| 用途            | 候補                   | 選定理由                    |
| ------------- | -------------------- | ----------------------- |
| テキスト → ベクトル変換 | Voyage AI            | Anthropic公式推奨。無料枠2億トークン |
| ベクトル保存・検索     | Pinecone Serverless  | フルマネージド。無料枠2GB          |
| 質問応答・文章生成     | Claude (`claude -p`) | サーバーにインストール済み           |

## エンジニアに相談したいこと

- 現在のノートのデータ管理の仕組みを踏まえて、ノート検索の仕組みをどの程度の手間で実現できるか
- チャットUIの追加は既存のフロントエンドにどう組み込むのが自然か
- ブロック編集中のAIプロンプトモード（半角スペース起動）は既存のエディタの仕組みと干渉しないか
- `claude -p`の起動レイテンシ（数秒）がUX上許容できるか、または軽減する方法があるか
- 技術選定や設計はエンジニア側の判断に任せたい。候補として挙げたサービスより良い方法があれば提案してほしい
